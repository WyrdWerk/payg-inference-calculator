# Phase 1 — PAYG Inference Calculator Foundation

## Phase Summary

Initial phase of the PAYG Inference Calculator (later renamed TokenWatch): a zero-dependency static site comparing pay-as-you-go LLM API pricing across providers. Work covered codebase review, data deduplication, Cloudflare Pages deployment with CI/CD, and a frontend redesign with percentage-based token input, org extraction, and an "All models" dropdown.

## What Was Built

- **Data pipeline**: Node ESM fetch script pulling `/v1/models` from six providers, normalizing pricing to $/M tokens, writing `pricing.json`
- **Deduplication**: Removed LLMGateway mirror provider and all `:free` entries; filtered negative placeholder prices from OpenRouter meta-routers
- **CI/CD**: GitHub Actions daily cron (`0 0 * * *`) to refresh pricing and deploy to Cloudflare Pages
- **Frontend redesign**: Replaced four absolute token fields with total-tokens + percentage breakdown (input, cached input, output)
- **Org extraction**: Added `org` field to model records from model ID prefix (e.g., `anthropic/claude-sonnet-5` → `org="anthropic"`), with cross-referencing for models without org prefixes and alias normalization (deepseek-ai→deepseek, zai-org→z-ai, meta-llama→meta)
- **"All models" option**: First option in model dropdown to view all models across providers without selecting a specific one
- **Security**: HTML injection fix in `renderTable` via `esc()` function

## Key Decisions

- Drop LLMGateway provider entirely (mirror of OpenRouter at identical prices, zero unique value)
- Drop all `:free` entries
- Filter negative placeholder prices from OpenRouter meta-routers (-1000000)
- Deploy via Cloudflare Pages instead of GitHub Pages
- Change refresh schedule from hourly to daily
- Replace absolute token inputs with total-tokens + percentage breakdown
- Add org column showing underlying model creator (OpenRouter treated as aggregator, not provider)
- Remove Cache Write column (not exposed in percentage model)
- Drop cache_write column from results table

## Technical Discoveries

- **Overlap analysis**: 724 model entries — 15 same-provider duplicates (OpenRouter `:free` vs paid pairs), 86 identical-price overlaps between LLMGateway and OpenRouter. Same model across different providers is intentional; same provider+model duplicates must be removed.
- **Canonical org resolution**: Build `canonical→org` map from models that have org prefixes; use cross-referencing for models without slash prefixes.
- **Cloudflare deploy**: `wrangler-action@v3` needs `--commit-dirty=true` when `pricing.json` is regenerated in-place during CI.
