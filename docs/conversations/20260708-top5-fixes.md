# TokenWatch — Top 5 Fixes (Canonicalization Unification, Video Audio Filter, Cache-Busting Automation, Test Suite)

**Session date**: 2026-07-08
**Live site**: https://tokenwatch.wyrdwerk.com

## Overview

This session addressed the top 5 verified issues identified in a full codebase audit. Each issue was confirmed against live code before fixing, and each fix includes a regression test.

1. **Canonicalization divergence** — the API's `normalizeId` used a greedy `-preview-.*$` catch-all that over-stripped `-preview-customtools`, causing distinct models (e.g. `gemini-3.1-pro` vs `gemini-3.1-pro-preview-customtools`) to collide in `/models/:id/providers`.
2. **Duplicated, diverged utilities** — `fetch-pricing.mjs` re-declared the entire `lib.mjs` surface inline (zero imports), and the two copies had drifted (`ORG_ALIASES` 11 vs 17 keys; `PROVIDER_NAME_MAP` 49 vs 55 keys).
3. **Video audio filter** — `String(null) === "null"` matched neither `"true"` nor `"false"`, dropping 22 of 38 pricing entries (58%) — including 8 entire models (Sora 2 Pro, Grok Imagine, Wan 2.6/2.7, Hailuo 2.3) — under either audio filter.
4. **Manual cache-busting** — 6 hand-edited `?v=` strings across 3 HTML files, no automation. Already caused one documented "stale cache masquerading as code bug" (2026-07-05 session log Lesson #7).
5. **Zero automated tests** — no test files, no `test` script, no test framework, CI ran no verification.

## 1. Canonicalization Unification (fixes #1 + #2)

### The Problem

Three copies of model-ID canonicalization existed:
- `scripts/fetch-pricing.mjs` — inline `canonicalId` (specific date patterns, preserves unknown `-preview-<foo>`)
- `scripts/lib.mjs` — exported `canonicalId` (byte-identical to fetch-pricing's)
- `functions/api/v1/[[route]].js` — local `normalizeId` (greedy `-preview-.*$` catch-all that over-stripped)

The API's version was the outlier. A request to `/api/v1/models/gemini-3.1-pro/providers` returned 3 merged models (pro + pro-preview + pro-preview-customtools) instead of just the 2 that legitimately canonicalize to `gemini-3.1-pro`.

### The Fix

Extracted the pure helpers into `shared/normalize.mjs` — a new module with **zero `node:` imports** so it bundles cleanly into the Cloudflare Worker without needing `nodejs_compat`. Both the Node pipeline (via `scripts/lib.mjs` re-export) and the API (direct import) now use the same source of truth.

**Key decision: `shared/` directory, not `scripts/`.** The blocker was that `scripts/lib.mjs` has a top-level `import { readFile } from 'node:fs/promises'` (used by `checkCoverageDrop`). If the Function imported from `lib.mjs`, esbuild would pull in the `node:fs` import, which breaks the Workers bundle unless `nodejs_compat` is enabled (no `wrangler.toml` exists today). By keeping `shared/normalize.mjs` pure (no `node:` imports), the cross-directory relative import `../../../shared/normalize.mjs` bundles correctly with no config changes. Verified against Cloudflare docs: Pages Functions are esbuild-bundled and follow relative imports.

**What moved to `shared/normalize.mjs`:** only `canonicalId` and `orgLookupKey` (the truly pure, dependency-free functions that actually diverged). `normalizeProvider`, `dedupKey`, `dedupModels` stay in `lib.mjs` (they depend on `PROVIDER_NAME_MAP`, which is data that belongs in lib alongside the other data tables).

**`fetch-pricing.mjs` consolidation:** deleted ~250 lines of inline re-declarations (`num`, `perTokToPerM`, `centsToDollars`, `passthrough`, `NON_TEXT_ID`, `isTextModel`, `ORG_ALIASES`, `orgFromId`, `orgFromName`, `canonicalId`, `orgLookupKey`, `PROVIDER_NAME_MAP`, `normalizeProvider`, `dedupKey`, `dedupModels`, `fetchJson`, `fetchJsonWithRetry`, inline `checkCoverageDrop`). All imported from `./lib.mjs` now. The `prevCount` return from `checkCoverageDrop` is captured to preserve the dry-run delta printout.

### Verification

- Full pipeline ran successfully (920 models, +10 from previous 910 — normal drift, no dedup regressions from the `ORG_ALIASES` merge).
- `/api/v1/models/gemini-3.1-pro/providers` now returns 2 matches (pro + pro-preview, both canonicalize to `gemini-3.1-pro`). The `gemini-3.1-pro-preview-customtools` variant is correctly a separate entry.
- Regression test in `test/canonicalization.test.mjs` asserts `-preview-customtools` is preserved.

## 2. Video Audio Filter Fix (#3)

### The Problem

`video-app.js:172` did `String(p.audio) !== state.audioFilter`. `String(null) === "null"` matched neither `"true"` nor `"false"`, so 22 of 38 pricing entries (58%) — including 8 entire models with only null-audio entries — vanished under either filter.

### The Fix (chosen semantics: null → "Without audio")

"With audio" = strictly `audio === true`; "Without / no audio" = anything not strictly true (null + false both survive). All 8 null-only models stay visible under "Without / no audio".

```js
// Before:
if (state.audioFilter !== '' && String(p.audio) !== state.audioFilter) continue;

// After:
if (state.audioFilter === 'true' && p.audio !== true) continue;
if (state.audioFilter === 'false' && p.audio === true) continue;
```

Also relabeled the HTML option from "Without audio" to "Without / no audio" for clarity, and updated the results-title copy.

### Verification

All 22 null-audio entries survive "Without / no audio". All 8 null-only models (Sora 2 Pro, Grok Imagine, Wan 2.6/2.7, Hailuo 2.3, HappyHorse 1.0/1.1, Kling Video O1) are now visible. Regression test in `test/video-audio.test.mjs`.

## 3. Cache-Busting Automation (#4)

### The Problem

6 hand-edited `?v=20260707e` strings across `index.html`, `image.html`, `video.html` (2 each: `styles.css` + the page's app JS). No automation. The 2026-07-05 session log documented a "stale cache masquerading as code bug" where returning visitors got old 9-cell JS against new 10-column HTML.

### The Fix (chosen approach: deploy job, no commit)

New `scripts/bust-cache.mjs`:
- Reads each `public/*.html`, finds `href="...?v=..."` and `src="...?v=..."` references.
- Computes an 8-char SHA-1 content hash for each referenced asset.
- Rewrites `?v=<old>` with `?v=<hash>` in place.
- Prints a summary.

Runs in CI:
- **`deploy` job** (push to main): after checkout + setup-node, before `wrangler pages deploy`.
- **`refresh` job** (daily cron): after the JSON commit step, before `wrangler pages deploy`.

The busted HTML is uploaded to Cloudflare but **not committed** — the repo keeps its old `?v=` strings. No commit noise, no forgotten bumps.

### Verification

Ran locally: 6 tokens busted across 3 HTML files. `styles.css` gets the same hash in all 3 pages (correct — shared asset). Each app JS gets its own hash. Repo HTML reverted after testing.

## 4. Test Suite (#5)

### The Problem

Zero test files, no `test` script, no test framework. CI ran no verification. This is what made issues #1–#4 possible.

### The Fix

Used Node's built-in `node:test` runner (stable since Node 18, CI uses Node 22) — zero dependencies, no install step. Consistent with the repo's zero-dep convention.

**4 test files, 61 tests total:**

- `test/canonicalization.test.mjs` (27 tests) — `canonicalId`/`orgLookupKey` behavior: org/ prefix strip, `:free`/`:thinking` strip, date suffixes, known preview-date patterns, **`-preview-customtools` preserved** (the regression), quantization preserved, `orgLookupKey` strips quant that `canonicalId` keeps.
- `test/parity.test.mjs` (3 tests) — loads real `public/pricing.json`, asserts the gemini-3.1-pro family has distinct keys, every model produces a non-empty key, glm-5.2 quant variants stay distinct.
- `test/api.test.mjs` (24 tests) — API routing + filters + sort with mocked `env.ASSETS` (small fixtures in `test/fixtures/`). Covers CORS, `/`, `/stats`, `/models` (all filters: org/provider/zdr/promo/sub/search/sort/order/limit/offset/min_context), `/models/:id/providers` (the regression fix, mix-aware sort, 404), `/orgs`, `/providers?zdr=true`, `/images/:id`, `/videos/:id`, 404. Documents the silent-fallback behavior for invalid sort keys.
- `test/video-audio.test.mjs` (5 tests) — regression for #3. Reproduces the fixed filter logic (video-app.js is a non-module global script, can't import directly). Asserts null-audio entries survive "Without / no audio", null-only models stay visible, flagships present.

**CI integration:** new `test` job runs on push/PR. `deploy` job gated via `needs: test`. `refresh` job runs tests before fetching.

### Verification

`npm test` → 61 pass, 0 fail.

## Key Decisions

1. **`shared/normalize.mjs` over splitting `lib.mjs`** — a new top-level `shared/` dir is cleaner than reorganizing `lib.mjs` internals. The purity constraint (no `node:` imports) is enforced by the module's purpose.
2. **Only `canonicalId` + `orgLookupKey` moved to shared** — `normalizeProvider`/`dedupKey`/`dedupModels` stay in `lib.mjs` because they depend on `PROVIDER_NAME_MAP` (data). The API doesn't need them anyway — it only needs `canonicalId`.
3. **Null → "Without audio"** (not a 4th "Audio N/A" option) — smallest diff, matches user expectation that "Without audio" shows models that don't have audio. Slight semantic conflation (null ≠ explicitly no-audio) but the catalog is small and the UX is clearer.
4. **Cache-bust in deploy job, no commit** — repo HTML stays at old `?v=` (always "dirty" on this dimension) but every deploy gets fresh content hashes. No commit noise, no forgotten bumps. The `refresh` job also busts before its deploy.
5. **`node:test` over Jest/Vitest** — zero-dep convention. No install step in CI. Stable since Node 18.
6. **Video filter test reproduces logic rather than importing** — `video-app.js` is a non-module global script. Refactoring it into an exported module is a follow-up, out of scope here. The test has a sync-note comment.

## Technical Discoveries

- **Cloudflare Pages Functions ARE esbuild-bundled** and follow relative imports outside `functions/`. No hard directory boundary. But `node:fs` in the imported module breaks the bundle unless `nodejs_compat` is enabled — confirmed via Cloudflare docs, SO, and GitHub issues.
- **`String.replace` with an async callback doesn't await** — `bust-cache.mjs` uses a `for...of` loop over `matchAll`, awaits all hashes, then does a synchronous replace with a Map.
- **`node --test test/` doesn't work in Node 22** — needs `node --test test/*.test.mjs` (explicit glob). Fixed in `package.json` and CI.
- **The `[[route]].js` had a local variable `canonicalId`** (the URL path segment) that shadowed the imported function. Renamed to `requestedId` to avoid the collision.
- **`fetch-pricing.mjs`'s `OR_MAX_RETRIES`/`OR_RETRY_DELAY_MS` constants** were only used by the inline `fetchJsonWithRetry`. Lib's version takes parameter defaults `(url, retries=1, delayMs=2000)` — equivalent. Constants deleted.
- **`checkCoverageDrop` return value** — lib returns `prevCount` (null on ENOENT); fetch-pricing's inline version didn't return it. The dry-run delta printout at lines 1040-1046 reuses `prevCount` — captured the return value to preserve that feature.
